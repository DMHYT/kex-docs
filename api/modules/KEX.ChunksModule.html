addLoadedChunk

  • addLoadedChunk(dimension: WorldOrDimension, x: number, z: number, name?: any_string): LoadedRectangle
    • Defined in kex.d.ts:3177

    Adds new rectangle-shaped ticking area to the given dimension, covering only one chunk on given coordinates. It is actually a shortcut, which looks like this:

    return addLoadedRectangle(dimension, name)
        .start(x, z)
        .end(x, z);
    since

    3.0

    Parameters

    • dimension: WorldOrDimension

      dimension where to create the ticking area

    • x: number
    • z: number
    • Optional name: any_string

      name of the area, that must be unique for the given dimension. If you omit this parameter, the name will be set to default ("Area{N}"), where {N} is an incrementing integer).

      • Note: before creating a new area with the name specified, ensure that there is no area with your name in the given dimension, using ChunksModule.areaExists. If you skip this check, you may get a java.lang.IllegalArgumentException when calling LoadedRectangle.load.

    Returns LoadedRectangle

    object to load the created area. This method returns the same object as addLoadedRectangle, but in this case you don't need to specify start and end position of the area.
